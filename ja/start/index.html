5分で分かる：idk-lamp の使い方

idk-lamp は、AI が「判断を出さない」べき地点で点く、静かな合図です。
エラーでも、失敗でもありません。判断を人間に返すための“境界信号”です。

Install：貼って使う（CDN）

“まず置いてみる” ための最小導入です。サイトやPoCにそのまま使えます。

1) CDN を読み込む

<script src="https://idk-lamp.org/dist/idk-lamp.min.js"></script>

2) タグを置く

<idk-lamp state="idk" label="Need human decision"></idk-lamp>

3) 見た目を CSS 変数で調整する（例）

:root { --idk-lamp-size: 84px; --idk-lamp-glow: 1; --idk-lamp-label-opacity: 0.9; }

※ state / theme / 変数名は、最新の仕様を idk-lamp リポジトリ で確認してください。

Gate：点けどころを決める

idk-lamp は “合図” です。
合図が生きるためには、いつ点けるか（＝AIに決めさせない条件） が先に必要です。

predictability-gate が扱うこと

• 予測/推論/生成の “不確かさ” を、運用上の判断に落とす
• 判断保留にする条件（誰が・いつ・何を確認するか）を定義する
• “止める” がエラーではなく、正常動作であることを揃える

predictability-gate を読む
推奨：Gate 側で “STOP/DEFER” を返したときに、UI 側で state="idk" を点灯する。
これで「止める → 人に返す」が一貫した体験になります。

FAQ

これは「警告灯」ですか？

いいえ。idk-lamp は “危険” を示すためではなく、判断を返すための合図です。
アラートの代替ではなく、「責任の境界」を UI 上で可視化します。

なぜ Demo / Repo / Gate を分けたままにするのですか？

役割が違うからです。
Demo は体験、idk-lamp repo は配布物、predictability-gate repo は運用判断。
分けたまま、サイトで導線だけ一本化するのが最も壊れません。

最初に何を見ればいいですか？

迷ったら Demo → このページの Install → Gate の順です。
“置ける” を先に作ると、Gate の議論が現実に降ります。
